<location>

  • Summary of Requirements for <location>
    The DLF/Aquifer Implementation Guidelines for Shareable MODS Records require the use of at least one <location> element with the <url> subelement. One and only one <location><url> subelement is required to have the usage attribute value "primary display". This element is repeatable. (page 75)
  • Per the Levels of Adoption Document Level 1: one or more <location> elements with a <url> subelement, one of which should point to the resource with its contextual material
  • Per the Levels of Adoption Document Level 2: Use of access attribute on location/url, or include only one URL that points to a view of the resource in context
  • Per the Levels of Adoption Document Level 4: one <location> element with one and only one <url> subelement contain the @usage attribute value "primary display"
  • Per the Levels of Adoption Document Level 4: @authority attributes in general
  • Per the Levels of Adoption Document Level 5: @access attribute on <location> subelement <url>
  • Per the Levels of Adoption Document Level 5: @authority attribute on <location> subelement <physicalLocation>, when applicable
  • Click on the Add One Location Url button to add another set of location input tags.
