    The first duty on the list involves building graphical user interfaces for teaching purposes. My master's project was PacketBuilder, a Java GUI application. Unfortunately, this Java code violates security on browsers and cannot go online, so I rewrote the code in Javascript, just so I could include it in this job application. PacketBuilder is an interactive teaching tool that I developed from scratch. It can be used as a toy, as an introduction to internet packets, or as a reference for upper level students in network programming.

    PacketBuilder is available at:

           http://www.cs.utk.edu/~deane/plumbium/may99/PB.html

    The second duty is to design and produce print-based materials. When I was a teaching assistant for the introductory course CS100, I designed and wrote the CS100 Lab Manual for Dr. David Straight. I used LaTex for digital layout and character control. This manual was developed over the course of a year, from my notes. Each semester it is updated slightly, but the basic manual that I created remains the same. I include the title page, table of contents, and one sample chapter with this letter. [None of these need to be returned.] Since the manual is in use this semester, I cannot put it online.

    The third duty on the list is to "provide user-centered graphic design..." I designed many, many, online tools for the CS100 course, but the enduring favorite has been:

           The CS100 Guide to How to Make Your Own Home Page!
           http://www.cs.utk.edu/~cs100/homepage_guide/AB_guide.html

    This guide was written for absolute beginners and every semester that I was a teaching assistant in CS100, many students came and thanked me for helping them get started with a homepage. I broke the guide into 6 short chapters. Each chapter delivers a "tangible product" that the student may add to his homepage.
