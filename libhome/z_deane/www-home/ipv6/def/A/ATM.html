Asynchronous Transfer Mode (ATM)

A standard which defines high-load, high-speed (1.544Mbps through 1.2Gbps), fixed-size packet (cell) switching with dynamic bandwidth allocation. ATM is also known as "fast packet." (From RFC1983.)

In this method of data transmission, the data is packed into packets called cells and a virtual circuit is established so that all of the cells follow the same path. Each cell has the same size (53 octets = 53x8 bits). There is no "ack" with ATM, so there is no automatic mechanism to resend a packet if an error is detected. ATM is used on B-ISDN.

More information at:

RFC2022: Support for Multicast over UNI 3.0/3.1 based ATM Networks.

RFC1954: Transmission fo Flow Labelled IPv4 on ATM Data Links Ipsilon Version 1.0, May 1996.

RFC1946: Native ATM Support for ST2+, May 1996.
