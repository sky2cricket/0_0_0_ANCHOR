Canonical Ordering of (0+1)*

Consider the set of Natural Numbers in binary.
The canonical order for this set is: 0, 1, 10, 11, 100, 101,....N

Consider the set (0+1)*.
The canonical order for this set is: lambda, 0, 1, 00, 01, 10, 11, 000, 001,....N

If you list the Canonical Ordering of (0+1)* and call each member of the list wi, where i is an integer, you can map each wi on the list to its corresponding index i.

From the above table, you can see the relationship between wi and the index i.
The relationship is:

Given wi:
the value for index i may be determined by adding a 1 to the right end of the string wi.

Conversely, given the value for index i:
the value for string wi may be determined by chopping off the rightmost 1 from the string i.

Where is Canonical Ordering applied?

In Chapter 8 of Hopcraft and Ullman, they start talking about Turing Machine Encodings as binary numbers. See the TM Encoding Page for a detailed explanation of how this encoding works.

There are several different ways of viewing the binary encoding of Turing Machines, but Dr. Langston favors the following:

ALL UNSIGNED POSITIVE BINARY NUMBERS CORRESPOND TO TURING MACHINES. This means TMs are countable, because they can be put in one-to-one correspondence with the set of Natural Numbers.

A Turing Machine Mi corresponds to the integer i.

Then Hopcraft and Ullman define two languages Ld and barLd (complement of Ld).

Ld = { < Mi, wi > | Mi not accept wi }

barLD = { < Mi, wi >| Mi accept wi }

You have to be able to decode Mi and wi to begin to have a clue what they are talking about!
