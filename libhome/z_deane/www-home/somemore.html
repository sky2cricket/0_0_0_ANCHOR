server side javascript
Server Side Javascript (SSJS) This method uses a set of Javascript database objects that are available on the web server. With this method, you develop in the same language for both the client side and the server side. This is a big advantage. It makes your code much simpler, easier to understand and more powerful. You can select which parts of your Javascript run on the client and which on the server. The Javascript database objects are very powerful and give you detailed control over your data (cursors) and database connections. SSJS requires that you compile your HTML files into a '.web' file and restart the application on the web server. This is a few extra steps than the template approach but with the clever use of hot keys, does not add much to development time. It results in a faster running application, so for systems that demand high performance, the extra steps are worth it. There are only a handful of ISP's that support SSJS. It requires Netscape web server. SSJS handles client state for you. You can select a method, such as client cookie, then create a client object in your application. As a given user goes from page to page, the client object is transparently maintained. If you have ever tried creating code that writes cookies or passes client ID's in a URL, you know what a pain that can be. For complex, high end systems, this is the best solution.
