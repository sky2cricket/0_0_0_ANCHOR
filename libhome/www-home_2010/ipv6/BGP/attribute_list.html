BGP Attribute List

This file is at http://www.cs.utk.edu/~deane/ipv6/BGP/attribute_list.html
Currently in use attribute type codes are listed below:

Value Mandatory? Code Meaning Reference
1 YES ORIGIN Identifies the AS that originated the associated routing information. [1]
2 YES AS_PATH Identifies the ASs through which routing information carried in this UPDATE has passed. [1]
3 NO NEXT_HOP Defines the IP address of the border router that should be used as the next hop to the destinations listed in the UPDATE message. [1]
4 NO MULTI_EXIT_DISC Used on external (inter-AS) links to discriminate among multiple exit or entry points to the same neighboring AS. [1]
5 NO LOCAL_PREF Included in all UPDATE messages that a given BGP speaker sends to the other BGP speakers located in its own AS to help in the Decision Process. [1]
6 NO ATOMIC_AGGREGATE Used to help sort out overlapping routes when there is a question about choosing a more specific versus a less specific route. [1]
7 NO AGGREGATOR An optional transitive attribute which may be included in UPDATES which are formed by aggregation. [1]
8 NO COMMUNITY An optional transitive attribute which may be used to control which routing information is accepted, prefered, or distributedto other neighbors. [2]
9 NO ORIGINATOR_ID Non-transitive attribute created by a route reflector that carries the ROUTER_ID of the originator of the route in the local AS. [3]
[3a]
10 NO CLUSTER_LIST Non-transitive attribute containing a sequence of CLUSTER_ID vlaues representing the reflection path that the route has passed. [3]
[3a]
11 NO DPA Destination Preference Attribute. (This is from a work in progress.) [4]
12 NO ADVERTISER Non-transitive attribute that defines the identifying address of the border router which originally submitted the route to a router server in order for it to be relayed to other Route Server Clients. [5]
13 NO RCID_PATH Non-transitive attribute that is composed of a sequence of RS Cluster Identifiers (RCID) that identifies each RS Cluster through which routing information carried in the UPDATE message has passed. [5]
...255 ... (reserved) reserved for future attributes ...
